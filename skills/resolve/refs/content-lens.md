# Content Lens

Additional questions for the five lenses when the work involves content, docs, or copy.

## Lens 1 Additions: Is This Real?

- Is the content actually wrong, or is the reader's context different than expected?
- Is this a gap in the content, or should the reader be directed elsewhere?
- Is the complaint about accuracy, clarity, or discoverability?

## Lens 2 Additions: What's Actually Going On?

- Is the information correct but poorly structured?
- Is the audience assumption wrong? (writing for experts, read by beginners)
- Is the content stale because the source of truth changed but docs weren't updated?

## Lens 3 Additions: What Else Can We Fix?

- Are other docs making the same assumption about audience?
- Is there a pattern of content going stale? Can we automate freshness checks?
- Are related pages consistent in terminology, tone, and structure?

## Lens 4 Additions: Should We Rethink?

- Should this be a tutorial instead of reference docs (or vice versa)?
- Should this content be generated from code/schema instead of hand-written?
- Is the information architecture putting this in the wrong place?

## Lens 5 Additions: Better Way?

- Can we link to existing content instead of writing new?
- Can we use a diagram instead of three paragraphs?
- Can we add an example instead of more explanation?
