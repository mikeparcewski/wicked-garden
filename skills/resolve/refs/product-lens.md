# Product Lens

Additional questions for the five lenses when the work involves features, user needs,
prioritization, or product strategy.

## Lens 1 Additions: Is This Real?

- Does anyone actually want this? What's the evidence of demand?
- Is this one user's request or a pattern across many?
- Is the pain real and frequent, or mild and rare?
- Would users notice if we did nothing?
- Are we building this because a stakeholder asked, or because users need it?

## Lens 2 Additions: What's Actually Going On?

- Is the user struggling with our product, or with the underlying problem?
- Is the feature request actually a workaround for a missing capability?
- Is the complaint about the feature, or about how they discovered it?
- Are users asking for what they want, or describing what they think the solution is?
  (What's the job-to-be-done behind the request?)

## Lens 3 Additions: What Else Can We Fix?

- Are there related user pain points in the same workflow?
- Would improving onboarding/docs reduce the need for this feature?
- Are there power-user workarounds that indicate a broader need?
- Can we improve the existing feature instead of adding a new one?

## Lens 4 Additions: Should We Rethink?

- Should this be a configuration option instead of a feature?
- Should this feature exist at all, or does it add complexity without proportional value?
- Would simplifying the product (removing features) solve the underlying confusion?
- Is this the right product to solve this problem, or should it be a separate tool?

## Lens 5 Additions: Better Way?

- Can we solve this with better defaults instead of a new setting?
- Can we solve this with documentation/education instead of code?
- Can we let users solve it themselves with an extension/API/integration?
- Is there a 10% effort solution that delivers 80% of the value?
- Can we validate the approach with a prototype before full investment?
