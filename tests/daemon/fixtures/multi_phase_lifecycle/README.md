Full clarify → design → build lifecycle with APPROVE at every gate and a final `wicked.project.completed` event. Each `wicked.phase.transitioned` event advances `current_phase` on the project row and sets the source phase to `state=approved` with `terminal_at` populated; the target phase gains `state=active` and `started_at`. After all three phases complete, `wicked.project.completed` sets `status=completed` and `current_phase=completed` on the project row. This fixture validates `current_phase` advances correctly through the chain, that every phase row ends with `state=approved` and a non-null `terminal_at`, and that the project reaches `status=completed`.
