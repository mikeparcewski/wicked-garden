# Onboarding Checklist

Template for comprehensive developer onboarding.

## Day 1: Environment Setup

### Prerequisites
- [ ] Development tools installed (IDE, Git, language runtime)
- [ ] Access to code repository
- [ ] Access to documentation
- [ ] Communication channels joined (Slack, Teams, etc.)

### Initial Setup
- [ ] Clone repository
- [ ] Install dependencies
- [ ] Configure local environment
- [ ] Verify build succeeds
- [ ] Run test suite successfully
- [ ] Start application locally

### Validation
- [ ] Made a trivial change (added comment)
- [ ] Tests still pass
- [ ] Application still runs
- [ ] Git workflow understood (branch, commit, push)

## Day 2-3: Codebase Exploration

### Structure Understanding
- [ ] Reviewed README and CONTRIBUTING docs
- [ ] Identified main source directories
- [ ] Understood project structure pattern (MVC, DDD, etc.)
- [ ] Located configuration files
- [ ] Found where dependencies are defined

### Entry Points
- [ ] Found application entry point (main file)
- [ ] Traced one request/execution flow end-to-end
- [ ] Identified where business logic lives
- [ ] Located database/persistence layer
- [ ] Found external service integrations

### Testing Strategy
- [ ] Located test directory
- [ ] Ran unit tests
- [ ] Ran integration tests (if applicable)
- [ ] Understood test file naming conventions
- [ ] Read tests for one component to understand behavior

## Day 4-5: First Contributions

### Bug Fix
- [ ] Found a "good first issue" or simple bug
- [ ] Reproduced issue locally
- [ ] Identified root cause
- [ ] Implemented fix
- [ ] Added test coverage
- [ ] Submitted pull request
- [ ] Addressed code review feedback

### Documentation
- [ ] Fixed typo or unclear documentation
- [ ] Added missing example
- [ ] Updated outdated information
- [ ] Clarified confusing section

### Code Quality
- [ ] Added test case for uncovered code
- [ ] Improved code comment
- [ ] Refactored small section (with tests)

## Week 2: Deeper Understanding

### Architecture
- [ ] Understand overall system architecture
- [ ] Know how components communicate
- [ ] Understand data flow
- [ ] Know performance considerations
- [ ] Understand security model

### Patterns and Practices
- [ ] Identified common patterns in codebase
- [ ] Understand error handling approach
- [ ] Know logging and monitoring practices
- [ ] Understand deployment process
- [ ] Know code review expectations

### Domain Knowledge
- [ ] Understand business domain
- [ ] Know key business terms
- [ ] Understand user personas
- [ ] Know critical user flows
- [ ] Understand success metrics

## Week 3-4: Ownership

### Feature Development
- [ ] Implemented small feature end-to-end
- [ ] Wrote comprehensive tests
- [ ] Updated documentation
- [ ] Deployed to staging/dev environment
- [ ] Verified in production (after merge)

### Collaboration
- [ ] Participated in code reviews
- [ ] Asked clarifying questions
- [ ] Shared knowledge with team
- [ ] Pair programmed with teammate
- [ ] Contributed to team discussion

### Independence
- [ ] Can work on tasks without constant guidance
- [ ] Know when to ask for help
- [ ] Understand team workflows
- [ ] Can estimate task complexity
- [ ] Contributing to sprint/milestone planning

## Ongoing Learning

### Continuous Improvement
- [ ] Read code in areas you haven't worked in
- [ ] Attend architecture discussions
- [ ] Review design documents
- [ ] Learn from production incidents
- [ ] Mentor newer team members

### Skill Development
- [ ] Learn unfamiliar parts of tech stack
- [ ] Understand performance profiling
- [ ] Learn debugging techniques
- [ ] Understand deployment pipeline
- [ ] Learn monitoring and alerting

## Personalization

Adapt this checklist based on:
- **Project complexity**: More time for complex systems
- **Your background**: Skip familiar areas, focus on gaps
- **Team expectations**: Align with team velocity
- **Project needs**: Prioritize high-impact areas
