## Root Cause

domain_adapter.query() returns ContextItems tagged with source="crew" or
source="jam" (see _DOMAIN_QUERIES in scripts/smaht/adapters/domain_adapter.py,
labels on lines 59 and 72). The old code filtered `if source != "brain": continue`,
which always dropped every item returned by domain_adapter. result was always [].

brain_adapter.query() (scripts/smaht/adapters/brain_adapter.py:195) returns
ContextItems tagged with source="brain" (set at line 244 of brain_adapter.py).

## Fix

Before (broken):
    from adapters import domain_adapter
    items = await domain_adapter.query(task)
    for item in items:
        if getattr(item, "source", "") != "brain":
            continue   # <-- always dropped; domain_adapter never emits source="brain"

After (correct):
    from adapters import brain_adapter
    items = await brain_adapter.query(task)
    for item in items:    # no source filter needed; brain_adapter always emits source="brain"

## Expected behavior change

Before fix:
  gather_memories("crew gate workflow") → [] (always, regardless of brain content)

After fix:
  gather_memories("crew gate workflow") → N items if wicked-brain FTS5 index
  has matching content for the query; [] only when brain is unreachable or
  query returns no results (graceful degradation unchanged).
