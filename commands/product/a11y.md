# /wicked-garden:product:a11y

Accessibility audit for UI code and components. Checks WCAG 2.1 Level AA compliance,
keyboard navigation, screen reader support, color contrast, and semantic structure.

## Usage

```bash
# Audit a component
/wicked-garden:product:a11y src/components/LoginForm

# Audit a directory
/wicked-garden:product:a11y src/components/

# Target specific WCAG level
/wicked-garden:product:a11y src/app --level AAA

# Quick scan for critical issues only
/wicked-garden:product:a11y src/components --quick
```

## Instructions

### 1. Parse Arguments

Extract `<target>`, `--level` (default: AA), and `--quick` flag.

### 2. Read Target

Read the target file(s). Focus on:
- HTML structure and semantic elements
- ARIA attributes and roles
- Interactive element patterns
- Form labels and error handling
- Color definitions for contrast analysis

### 3. Inline Audit

Perform the accessibility audit using the accessibility skill criteria:

**Perceivable**
- [ ] Images have alt text (descriptive or `alt=""` for decorative)
- [ ] Color not the sole information conveyor
- [ ] Text contrast ≥4.5:1 (normal), ≥3:1 (large text 18pt+)
- [ ] UI component contrast ≥3:1
- [ ] Content survives 200% zoom

**Operable**
- [ ] All interactive elements reachable via Tab
- [ ] Tab order follows visual/logical order
- [ ] No keyboard traps; modals have ESC exit
- [ ] Visible focus indicators on all interactive elements
- [ ] No content flashing >3×/second

**Understandable**
- [ ] `lang` on `<html>`
- [ ] Form errors identify field and explain fix
- [ ] Labels describe purpose (not just appearance)
- [ ] Consistent navigation patterns

**Robust**
- [ ] Valid HTML; no duplicate IDs
- [ ] ARIA only where native semantics insufficient
- [ ] ARIA roles/states/properties correctly used
- [ ] Dynamic content uses `aria-live`

### 4. Report Findings

```markdown
## Accessibility Audit: {target}

**WCAG Level Achieved**: {Partial A | A | Partial AA | AA | AAA}
**Score**: {1–5}
**Critical Issues**: {count}

### Compliance by Principle

| Principle | Status | Issues |
|-----------|--------|--------|
| Perceivable | ✓/⚠/✗ | {n} |
| Operable | ✓/⚠/✗ | {n} |
| Understandable | ✓/⚠/✗ | {n} |
| Robust | ✓/⚠/✗ | {n} |

### Issues

#### Critical (blocks access)
- **WCAG {criterion}**: {issue}
  - Location: {file:line}
  - Impact: {who is blocked}
  - Fix: {specific change}

#### Major
- {issue with recommendation}

#### Minor
- {enhancement}

### Passed Checks
- {what is accessible}

### Priority Actions
1. {must fix}
2. {should fix}
```

## Quick Mode

With `--quick`: return only Critical issues. Skip the full checklist.

## Integration
